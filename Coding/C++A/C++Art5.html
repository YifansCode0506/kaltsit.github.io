Remove Element

Publication Date: 2025-04-16

Problem Description

Given an array nums and a value val, remove all instances of that value in-place and return the new length of the array.

Do not allocate extra space for another array; you must perform this operation with O(1) extra memory.

The order of elements can be changed. You do not need to consider elements beyond the new length.

Example 1

Input: nums = [3,2,2,3], val = 3
Output: 2, and the first two elements of nums are 2 and 2.

Example 2

Input: nums = [0,1,2,2,3,0,4,2], val = 2
Output: 5, and the first five elements are 0, 1, 3, 0, 4.

Important Notes

It might seem natural to \"delete\" an element directly. However, in C++, array elements are stored in contiguous memory locations, so you cannot physically delete a single element. You can only overwrite existing elements.

Fundamentals of Array Theory

Brute Force Approach

The brute force method uses two nested loops: one to traverse the array and another to shift elements after encountering a value to be removed.

This approach has a time complexity of O(n^2) and can pass on platforms like LeetCode for small inputs.

C++ Code:


    // Time Complexity: O(n^2)
    // Space Complexity: O(1)
    class Solution {
    public:
        int removeElement(vector<int>& nums, int val) {
            int size = nums.size();
            for (int i = 0; i < size; i++) {
                if (nums[i] == val) {
                    for (int j = i + 1; j < size; j++) {
                        nums[j - 1] = nums[j];
                    }
                    size--;
                    i--;
                }
            }
            return size;
        }
    };
  • Time Complexity: O(n^2)
  • Space Complexity: O(1)

Two Pointers Approach

Two pointers method (fast and slow pointers): Use a fast pointer to find valid elements and a slow pointer to overwrite invalid ones.

Concept

  • Fast Pointer: Scans every element.
  • Slow Pointer: Marks the position to overwrite.

Many struggle with this idea initially, but understanding the role of each pointer makes the logic clear.

C++ Code:


    // Time Complexity: O(n)
    // Space Complexity: O(1)
    class Solution {
    public:
        int removeElement(vector<int>& nums, int val) {
            int slowIndex = 0;
            for (int fastIndex = 0; fastIndex < nums.size(); fastIndex++) {
                if (nums[fastIndex] != val) {
                    nums[slowIndex++] = nums[fastIndex];
                }
            }
            return slowIndex;
        }
    };

Note: This method does not preserve the relative order of elements, but that is acceptable based on the problem statement.

  • Time Complexity: O(n)
  • Space Complexity: O(1)

Project 1: Smart City Cyber Security Framework

With the rapid development of smart cities, more public systems (such as waste management, traffic control, water supply, and government services) rely on the internet and IoT technologies. However, these systems also face serious cybersecurity threats, such as hackers attacking traffic systems or stealing citizen data. Our team was tasked with designing a complete cybersecurity framework to protect critical infrastructure from attacks.

We first analyzed six key systems: smart waste management, electronic toll collection (ETC), government service apps, public Wi-Fi, bus tracking systems, and smart water management. These systems handle sensitive data, such as personal information, traffic records, and water quality data. We identified key protection priorities for each system, such as preventing data tampering, leakage, and unauthorized access.

We applied the STRIDE model to identify possible threats like spoofing, tampering, and denial of service attacks. This helped us understand potential attacker behaviors and design proactive defense strategies.

ğŸ” How did we detect vulnerabilities?

We conducted vulnerability assessments, covering software bugs (like SQL injections), configuration errors, and weak passwords. We also used penetration testing tools to simulate attacks, such as bypassing login authentication and intercepting unsecured Wi-Fi traffic.

We assessed all identified risks by analyzing their likelihood and impact, classifying them into high, medium, and low risk. For example, the government services platform was a top priority due to the large amount of sensitive data. Based on risk levels, we proposed four strategies: risk mitigation, transfer, acceptance, and avoidance.

Tools like Splunk, Metasploit, and SIEM were used to ensure security, fast recovery, and system optimization under attack scenarios.

Project 2: Python Schedule

ğŸ§­ Project Introduction and Main Functions

Click to view details

This is a command-line based schedule management tool allowing users to create, edit, view, and save personal events. It simulates a weekly timetable system, checking for event conflicts and supporting save/load operations.

Users input commands like:
- create: create new event;
- edit: edit existing event;
- table: view timetable;
- save/load: save or load database;
- quit: exit the program (prompting to save unsaved changes).

Great for practicing CLI input, flow control, and file operations.

ğŸ§± Data Structures and Time Processing

Click to view details

Events are stored in a list (db), with each event as a dictionary:

{
    'title': 'Meeting',
    'start': 540, // Sunday 9:00 AM (9*60 minutes)
    'end': 600,
    'location': 'Room 101'
  }

- Time is stored as minutes from Sunday 0:00;
- Users input week, hour, and minute via input_time();
- Time display uses str_time() and str_time24() for 12/24-hour formats.
- save_time() and load_time() handle persistence.

âœ… Validation and Editing Workflow

Event creation/editing includes validations:

  • Title cannot be \"TODO\".
  • Location cannot be \"Where\".
  • Start time must be before end time.
  • No overlapping events allowed.

On failure, prompts for re-entry. Actions during edit include Save, Quit, or Delete, recognized by input_cmd() (abbreviations supported).
